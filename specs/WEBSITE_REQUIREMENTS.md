# WEBSITE REQUIREMENTS

## Functional requirements

### MVP pages
- Home
- Services (combined overview)
- FAQ
- Contact (with file upload)
- Impressum
- Datenschutz
- AGB (initial version)

### Lead capture
- Form fields: name, email, optional phone, request description, file attachment(s).
- Consent checkbox linked to privacy policy.
- Submission routing to email/CRM.
- User receives acknowledgement confirmation.

### Content requirements
- German-first copy with clear, transcreated wording.
- Service benefits, turnaround expectations, and process clarity.
- Testimonials and/or trust markers.
- FAQ entries about formats, pricing logic, turnaround, revisions.

### SEO/GEO baseline
- Per-page title and description.
- Proper heading hierarchy.
- Image alt texts.
- `robots.txt` and `sitemap.xml`.
- Initial structured data support path.

## Non-functional requirements
- Mobile-first responsive layout.
- Accessibility baseline aligned with WCAG 2.2 fundamentals.
- Fast loading behavior and optimized assets.
- GDPR-aware forms and tracking readiness.

## Constraints
- Prefer existing design system components and tokens.
- Keep architecture extensible for calculators, resource center, and portal phases.
