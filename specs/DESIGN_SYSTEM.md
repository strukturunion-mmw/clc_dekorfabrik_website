# DESIGN SYSTEM IMPLEMENTATION NOTES

## Source
Primary source: existing design kit and website component patterns in this repository.

## Principles
- Consistent typography and spacing scale.
- Strong visual hierarchy with conversion-focused CTA placement.
- Reusable components over page-specific one-offs.
- Accessibility-first semantics and states.

## Component priorities (MVP)
- Header/navigation
- Hero section
- Feature/value cards
- CTA buttons and panels
- FAQ accordion
- Contact/upload form sections
- Footer/legal navigation

## Styling guidance
- Use centralized tokens and existing CSS architecture.
- Keep color/contrast combinations accessible.
- Preserve spacing rhythm between sections.
- Avoid introducing competing style conventions.

## Asset guidance
- Keep canonical logos/icons in `public/`.
- Use optimized formats where possible.
- Provide descriptive alt text and captions when context is needed.
